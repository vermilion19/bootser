## 작업

$ARGUMENTS로 지정된 기술 주제에 대해 주니어에서 시니어로 성장하기 위한 단계별 학습 경로를 생성합니다.

### 입력

- `$ARGUMENTS`로 전달된 기술 주제 또는 키워드
- 구체적인 기술명 (예: Kafka, Kubernetes) 또는 추상적 주제 (예: 대용량 트래픽 처리) 모두 가능

### 학습 경로 구성

#### Phase 1: 기초 (Foundation)
- 핵심 개념과 용어 정리
- 왜 필요한지 (문제 인식)
- 기본 동작 원리
- 간단한 로컬 환경 실습

#### Phase 2: 실전 적용 (Practice)
- 프로젝트에 적용하는 방법
- 주요 설정과 옵션
- 자주 쓰는 패턴과 안티패턴
- 트러블슈팅 기초

#### Phase 3: 심화 (Advanced)
- 내부 동작 원리 (Under the hood)
- 성능 최적화 전략
- 장애 대응 및 모니터링
- 엣지 케이스와 한계점

#### Phase 4: 운영 / 설계 (Production)
- 프로덕션 환경 구성 노하우
- 스케일링 전략
- 보안 고려사항
- 다른 기술과의 조합 및 아키텍처 설계

### 출력 형식

```markdown
## [기술 주제] 학습 로드맵

### 개요
- 주제: [기술명]
- 예상 학습 범위: Phase 1~4
- 선행 지식: [필요한 사전 지식]
- 관련 기술 스택: [연관 기술 목록]

---

### Phase 1: 기초 (Foundation)

#### 학습 목표
- 목표 1
- 목표 2

#### 핵심 개념
| 개념 | 설명 |
|------|------|
| 용어1 | 간단한 설명 |
| 용어2 | 간단한 설명 |

#### 학습 항목
- [ ] 1-1. 항목 제목
  - 설명
  - 실습: 구체적인 실습 내용
- [ ] 1-2. 항목 제목
  - 설명

#### 추천 리소스
- 공식 문서: [링크 또는 키워드]
- 추천 서적/강의: [제목]

---

### Phase 2: 실전 적용 (Practice)

#### 학습 목표
...

#### 학습 항목
- [ ] 2-1. 항목 제목
  - 설명
  - 실습: 현재 프로젝트(Booster)에 적용하는 시나리오

---

### Phase 3: 심화 (Advanced)

#### 학습 목표
...

#### 학습 항목
- [ ] 3-1. 항목 제목
  - 설명

#### 면접 빈출 질문
- Q: 질문
- Q: 질문

---

### Phase 4: 운영 / 설계 (Production)

#### 학습 목표
...

#### 학습 항목
- [ ] 4-1. 항목 제목
  - 설명

#### 실무 체크리스트
- [ ] 프로덕션 배포 전 확인 사항
- [ ] 모니터링 설정
- [ ] 장애 대응 플레이북

---

### 연관 학습 경로
- [관련 주제 1]: 이 주제를 학습한 후 추천
- [관련 주제 2]: 병행 학습 추천
```

### 작성 규칙

1. **프로젝트 연계**: 가능하면 현재 프로젝트(Booster)에 적용할 수 있는 실습 시나리오 포함
2. **체크리스트 형태**: 학습 진행 상황을 추적할 수 있도록 `- [ ]` 사용
3. **면접 연계**: Phase 3 이상에서 면접 빈출 질문 포함
4. **실용적**: 이론보다 실습과 적용 중심으로 구성

### 사용 예시

```
/roadmap Distributed Tracing (Zipkin/Sleuth)
/roadmap 고가용성 DB 클러스터링 구성 및 운영
/roadmap JVM 튜닝 및 메모리 프로파일링
/roadmap Kafka 심화 (파티셔닝, 리밸런싱)
/roadmap Spring Security + OAuth2
```
