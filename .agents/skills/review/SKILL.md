## 작업

현재 변경사항 또는 지정된 파일에 대해 코드 리뷰를 수행합니다.

### 리뷰 대상

- `$ARGUMENTS`가 없으면: `git diff`로 현재 변경사항 리뷰
- `$ARGUMENTS`가 파일 경로면: 해당 파일 리뷰
- `$ARGUMENTS`가 PR 번호면: 해당 PR 리뷰

### 리뷰 관점

#### 1. 버그 가능성
- NullPointerException 위험
- 경계값 처리 (0, 음수, 빈 컬렉션)
- 예외 처리 누락
- 동시성 이슈

#### 2. 성능
- N+1 쿼리 문제
- 불필요한 DB 호출
- 메모리 누수 가능성
- 비효율적인 알고리즘

#### 3. 보안
- SQL Injection
- XSS 취약점
- 민감정보 하드코딩
- 인증/인가 누락

#### 4. 설계/가독성
- SOLID 원칙 위반
- 과도한 복잡도
- 불명확한 네이밍
- 중복 코드

#### 5. 테스트
- 테스트 필요 여부
- 엣지 케이스 커버리지

### 출력 형식

```markdown
## 코드 리뷰 결과

### Critical (반드시 수정)
- [ ] 파일:라인 - 내용

### Warning (수정 권장)
- [ ] 파일:라인 - 내용

### Suggestion (선택적 개선)
- [ ] 파일:라인 - 내용

### Good (잘한 점)
- 내용
```

### 사용 예시

```
/review                           # 현재 git diff 리뷰
/review src/main/java/.../*.java  # 특정 파일 리뷰
/review 123                       # PR #123 리뷰
```
