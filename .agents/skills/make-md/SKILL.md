## 작업

직전에 내가 응답한 내용을 마크다운 파일로 정리하여 저장합니다.

### 파일 저장 규칙

1. **파일명**: $ARGUMENTS가 있으면 해당 이름 사용, 없으면 내용 기반으로 적절한 파일명 생성
2. **저장 위치**: 현재 작업 디렉토리 또는 `docs/` 폴더
3. **파일 확장자**: `.md`

### 정리 규칙

1. 제목(`#`)을 명확하게 작성
2. 섹션을 `##`, `###`로 구분
3. 코드는 적절한 언어 태그와 함께 코드 블록으로 감싸기
4. 표가 필요한 경우 마크다운 테이블 사용
5. 중요한 내용은 **굵게** 또는 `>` 인용문으로 강조
6. 불필요한 대화체 제거하고 문서 형식으로 정리

### 사용 예시

```
/make-md                     # 자동 파일명으로 저장
/make-md api-guide           # api-guide.md로 저장
/make-md docs/architecture   # docs/architecture.md로 저장
```
