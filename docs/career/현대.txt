[지원동기]
현대자동차의 커넥티드 서비스가 확대되면서 차량과 클라우드가 상시 연결되는 환경에서 백엔드 시스템의 중요성이 빠르게 커지고 있습니다. 이 환경에서는 작은 지연이나 장애도 곧 고객의 차량 경험에 직접적인 영향을 주기 때문에, 성능과 안정성을 기본값으로 만드는 백엔드의 역할이 특히 의미 있다고 느껴 지원했습니다.

저는 IoT 클라우드 플랫폼에서 다수의 디바이스가 연동되는 환경을 설계하고 운영하며 이 역할의 기반을 다져왔습니다. 트래픽 증가에 대응해 MQTT와 Redis Pub/Sub 클러스터링으로 수평 확장 구조를 만들었고, 결제 시스템에서는 비관적 락으로 동시성 이슈를 해결했습니다.
또한 대량 데이터 조회 시 커서 기반 페이징과 인덱스 튜닝으로 응답 속도를 개선했습니다.
특히 차량용 블랙박스 단말과 서버 간 통신을 개발하며, 펌웨어 제약에 따른 프로토콜 설계 등 일반적인 웹 백엔드와는 다른 문제들을 직접 해결했습니다.
 이 과정에서 하드웨어 팀과 협업하며 서버 관점의 요구사항을 조율하는 역량도 함께 키울 수 있었습니다.

입사 후에는 먼저 현대자동차의 서비스 도메인을 깊이 이해하고, 모니터링 체계 고도화와 장애 대응 프로세스 정착에 기여하겠습니다.
프론트엔드 및 모바일 개발자와의 API 계약을 명확히 하고, 병목이 되는 쿼리와 트랜잭션을 지속적으로 개선해 안정적인 서비스 품질을 확보하겠습니다.
장기적으로는 차량 단말과의 통신 경험을 살려, 글로벌로 확장되는 커넥티드 카 트래픽을 유연하게 수용할 수 있는 분산 아키텍처 설계에 참여하며 현대자동차의 모빌리티 서비스를 뒷받침하는 시니어 백엔드 엔지니어로 성장하고 싶습니다.

[직무 수행에 있어 가장 중요한(필요한) 역량]
IT Backend 직무 수행에 있어 가장 중요한 역량은 문제의 본질을 정확히 파악하고, 비즈니스 요구와 시스템 환경에 맞춰 적정 기술을 선택하는 문제 해결력이라고 생각합니다.

백엔드 개발자가 마주하는 문제는 표면적으로 비슷해 보여도 원인과 맥락에 따라 해결 방향이 완전히 달라집니다. 같은 성능 저하라도 쿼리 구조의 문제인지, 데이터 접근 패턴 자체가 설계와 맞지 않는 것인지에 따라 단일 쿼리 튜닝으로 끝날 수도 있고 시스템 구조를 분리해야 할 수도 있습니다.
같은 동시성 이슈라도 절대적인 정합성이 필요한 핀테크 결제에서는 DB의 비관적 락을 통해 원천 차단해야 하지만, 간헐적 유실이 허용되면서 즉시성이 중요한 실시간 센서 데이터 환경에서는 Redis Pub/Sub을 활용한 분산 처리 전략을 선택해야 합니다.
대용량 데이터 조회 시에도 기존 페이징의 한계를 인지하고 커서 기반(No-Offset)으로 전환했던 것처럼, 결국 핵심은 어떤 기술을 아느냐가 아니라 '지금 이 상황에서 왜 이 방식이어야 하는지'를 판단하는 것입니다.

대 AI 시대에 이 역량은 더욱 중요해진다고 생각합니다. AI 도구가 코드 생성 속도를 비약적으로 높여주면서, 코드를 작성하는 능력 자체의 희소성은 점점 낮아지고 있습니다.
반면 AI가 생성한 코드가 현재 시스템의 맥락에 적합한지, 성능과 안정성 측면에서 어떤 트레이드오프가 있는지를 판단하는 것은 여전히 사람의 몫입니다. 코드
를 빠르게 만드는 것보다 무엇을 만들어야 하는지, 왜 이 구조여야 하는지를 결정하고 책임지는 능력이 백엔드 개발자의 본질적 가치입니다.

현대자동차의 모빌리티 서비스 환경은 다양한 클라이언트의 요구와 방대한 차량 데이터가 교차하는 곳일거라 예상됩니다. 어떤 서비스를 맡게 되든, 문제의 표면이 아닌 근본 원인을 찾고 근거 있는 기술 판단을 내리는 개발자로서 현대자동차의 신뢰성 높은 백엔드 생태계를 구축하는 데 기여하겠습니다.
